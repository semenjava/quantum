# Data model

Authorization model:

  • unique ID in some format like CNA where C=some company ID, N=id of client within company, A=letter of alphabet increasing with each authorization for the client. For ex.: MSJR000004C
  • person that does the authorization
  • TBD "Presenting problem"
  • ICD-10 Diagnosis code (replaces "presenting problem")
  • TBD Auth O/C - never used!
  • Network (In/Out) - populated on EOB
  • TBD
  • Auth Units: up to 4 digits

# Authorization time period

# Setting an authorization time period

  • Authorizations can be backdated

  • End date of authorization should be calculated automatically by first setting the start date, entering the frequency, then clicking a "calculate" button

    • for example 1 x “initial evaluation” = one month auth

  • Frequencies should be configurable

# Authorization expiry

  • if period expired, authorization is voided

# Authorization CPT codes

  • Cpt codes and frequency is requested by providers
    • follows industry standards (intake staff know this)
  • One authorization can only have one CPT code

Interesting, but not directly relevant:

  • Quantum sees 65% utilization of authorization on average
  • Mistakes in data entry of CPT codes do not happen very often, so not a big concern.
