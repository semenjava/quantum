# Communication templates

Settings should include an CRUD list of all emails, text messages and fax cover letters sent out via the app, both automatic notifications and those sent manually by staff.

# Email

  • allow selection of one or multiple TO recipients
  • allow selection of zero, one or multiple each of CC, BCC recipients
  • offer programmatically defined recipient placeholders, such as "client", "provider", ...
  • subject line and body text. Both should support basic pattern substitution to insert a number of variables such as client.name, client.title, etc. as well as rich text formatting

# Text messages

  • TO recipient, same as in email
  • A plain text field with support for the basic pattern substitution (see email above)

# Fax

  • A plain text field with support for the basic pattern substitution (see email above)
  • The text will be integrated into the fax template layout (not necessarily as a separate page)

# Configurable codes & other value lists

There are various configurable lists of values in use throughout the application.

Some of these will be used in simple dropdowns, so they consist simply of one column (the value). In other cases they will be used in other more complex contexts, these also have a "description" column (such as with CPT or ICD-10 codes). Both share the same admin panel interface.

Deleting a value will actually only disable it from being selected as a new value. Any instances where that value has been previously assigned will remain unchanged. Importantly, even if that record is edited and saved, the value should not be changed unless explicitly done so by the user. That means the deleted value will need to be shown in the dropdown (but of course only if it was previously selected)
