# Complex components

# Email templates, editor, sending

  • renders a template defined in Global Settings
  • allows editing of recipients (as simple text field), subject and body
  • has a "send me a copy" check box (for emails)
  • optionally allows selection of a different template
  • attach files

Auto reply on text message interface - "we do not monitor this, please contact xyz"

Keep log of all outgoing messages.

# Table component

A component we can use for all table data views. Features should include:

  • filtering by any column (dropdown or full-text search within)
  • sorting by any suitable column (alphabetically, descending or ascending)
  • selecting which columns to view via a list of check boxes
  • exporting data as CSV, XLSX
  • exporting data as PDF (automatically select portrait / landscape based on mumber of columns)

This component should be backed by an equally reusable backend function or controller that can be used for any data.

# Activity log

Mockups are part of the case diagram: https://whimsical.com/case-7QvwaV2fuGtVba8na3pehP (opens new window)

A log of activities that can be attached to related to various entities in the app. For example, there is an activity log for authorizations and claims, but also for providers, employers, ..

Some of the activity in the log will be saved automatically. For that, the log component exposes an API so that any action in the app can register log events.

Other activity will be logged manually, such as internal comments, phone calls, ...

The component should use the table component above, and allow for selecting one or multiple types of entitiy logs to be displayed at once, filtered as needed.

# Notifications

Staff may receive a variety of notifications.

Notifications can be:

  • a) temporary (displayed in-app but not persistent when page is reloaded)
  • b) persistent (displayed until marked as read)

In addition, each has one of the following types:

  • info
  • warning
  • error

# Notification popup

Notifications should be shown in app as small pills that can be closed as needed. Multiple notifications can stack on top of each other. Notifications should be pushed (web sockets) to or pulled by (polling) the client without page reload.

# Notifications list

Staff can review past persistent notifications in a table view in a separate main menu page.
